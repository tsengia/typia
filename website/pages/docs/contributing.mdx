## Contributing to Typia

Before Contributing to Typia, make sure to give the [`CONTRIBUTING.md`](https://github.com/samchon/typia/blob/master/CONTRIBUTING.md) file a read.

Basic steps to getting started with developing Typia:
1. Fork the repository on GitHub
2. Clone the repository
3. Install dependencies using: `npm install`
4. Checkout a new branch for your changes using: `git checkout -b your-new-feature-here`
5. Run the commands listed in the [`CONTRIBUTING.md`](https://github.com/samchon/typia/blob/master/CONTRIBUTING.md) to generate and run the test suite.

## Typia Goals
1. Speed
2. Correctness
3. For JSON related features, exact `JSON.stringify()` and `JSON.parse()` compatibility

## Typia's Repository Structure
The Typia repository has the following folder structure:
```
typia
├─── benchmark          Code for performance benchmarks
├─── build
├─── errors             Test cases that Typia explicitly lacks support for
├─── packages
├─── src                Source code for Typia
│    ├ executable       Code for the Typia CLI interface
│    ├ factories        Factory methods that produce Typescript code during transformation
│    ├ functional       Functional helper routines
│    ├ programmers      
│    ├ schemas
│    │ ├ json           Classes for Swagger/OpenAPI specifications
│    │ └ metadata       Classes that store information about the TypeScript AST during transformation
│    ├ tags             Code for validation tags
│    ├ transformers     Modules for performing the TypeScript transformation
│    ├ typings          Helper type definitions
│    └ utils            Helper functions
├─── test               Test suite source code
└─── website            MDX files for the typia.io website
```
